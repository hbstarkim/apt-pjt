# 아파트 평면도 비교 · 가구 배치 시뮬레이터

이사 시 두 집(현재 집 A / 이사갈 집 B)의 평면도를 나란히 띄워 놓고, 방 크기를 비교하며 동일한 가구를 각 방에 배치·시뮬레이션하기 위한 로컬 웹 앱입니다. 설치가 필요 없으며 크롬 브라우저에서 바로 실행됩니다. (오프라인 동작 — Konva.js 라이브러리를 로컬에 포함)

## 실행 방법

### 권장: 로컬 서버로 실행 (모든 기능 + OCR)
- `start.ps1` 을 우클릭 → "PowerShell에서 실행" 합니다. (Node.js 필요)
- 자동으로 서버가 뜨고 크롬에서 `http://localhost:8765/` 가 열립니다.
- OCR(사진 수치 자동인식)은 브라우저 보안 정책상 이 방식에서만 동작합니다.

### 간편 실행 (OCR 제외)
- `index.html` 파일을 더블클릭하거나 크롬으로 엽니다.
- 평면도·가구·충돌 경고·PNG/PDF 내보내기는 모두 동작하며, OCR만 제한됩니다.

> 처음 예시를 보려면 주소 끝에 `?demo=1` 을 붙입니다. (예: `http://localhost:8765/?demo=1`)
> 단, 두 방식은 저장소(localStorage)가 분리됩니다. 작업을 옮길 때는 **JSON 내보내기/불러오기** 를 이용하세요.

## 주요 기능

### 1. 평면도 입력
- 각 도면 툴바의 **+ 방** 으로 방 이름·가로·세로(현재 단위 기준)를 입력하면 사각형 방이 생성됩니다.
- **배경** 으로 평면도 사진을 업로드한 뒤 **축척보정** 으로 실제 길이를 아는 두 점을 클릭하고 실측 길이를 입력하면, 사진이 실제 비율로 맞춰집니다. 그 위에 방을 그대로 따라 그릴 수 있습니다. (사진의 치수를 자동으로 읽어 그리는 OCR 기능은 향후 확장 항목입니다.)

### 2. 가구 배치
- 좌측 **가구 라이브러리** 에서 이름·가로·세로·**높이**·색상을 입력해 가구를 등록합니다. (높이는 3D 보기에서 사용되며, 비워 두면 이름으로 추정한 기본값이 적용됩니다)
- **한 번 등록한 가구는 영구 저장**되어 다음에 다시 열어도, "초기화"하거나 다른 파일을 불러와도 목록에 그대로 남습니다(브라우저의 전용 저장소에 보관).
- **드래그 앤 드롭**: 목록의 가구를 도면 위로 끌어다 놓으면 그 자리에 배치됩니다. (방 비교창의 각 방에도 동일하게 드롭 가능)
- 또는 가구 항목의 **→A / →B**, 도면 툴바의 **+ 가구** 로도 추가합니다.
- 도형을 드래그해 이동하고, 선택 후 상단 회전 핸들로 **360° 자유 회전**(90° 자동 스냅)이 가능합니다.
- **다중 선택·이동**: 빈 곳을 클릭해 선택을 해제한 뒤, **Shift+클릭**(또는 Ctrl/⌘+클릭)으로 여러 도형을 함께 선택하고, 그중 하나를 드래그하면 **선택한 모든 도형이 함께 이동**합니다. 우측 속성창에서 한꺼번에 삭제할 수도 있습니다.
- 방은 모서리 핸들로 크기를 조절할 수 있고(가구는 실제 치수 고정), 우측 **속성** 패널에서 정밀하게 수정합니다.

> 참고: 가구 카탈로그는 실행 방식별 저장소(브라우저 origin)에 보관됩니다. `start.ps1`(http) 과 더블클릭(file) 을 섞어 쓰면 목록이 분리될 수 있으니, **한 가지 실행 방식**으로 사용하시길 권장합니다.

### 3. 두 집 / 방 비교
- 상단 **📐 방 비교** 를 누르고 도면에서 방 2개를 클릭하면 비교 팝업이 열립니다.
- 두 방이 **같은 축척**으로 나란히 표시되고, 면적·가로·세로 차이가 수치로 비교됩니다(증가는 초록, 감소는 빨강).
- **겹쳐보기** 로 한 방의 외곽선을 다른 방 위에 덧대 크기 차이를 직관적으로 확인합니다.
- 팝업의 **가구 트레이** 에서 동일한 가구를 각 방에 넣어 들어가는지 시뮬레이션합니다.

### 4. 가구 겹침 충돌 경고
- 가구를 드래그·회전하는 즉시 충돌을 검사합니다(회전 사각형 정밀 판정 — SAT).
- **겹친 가구는 빨강 점선**, **방 밖으로 벗어난 가구는 주황 점선** 으로 표시됩니다.
- 도면 툴바 오른쪽에 **⚠ 충돌 N개 · 방 이탈 M개** 배지가 실시간 갱신됩니다.

### 5. 사진 치수 자동인식 (OCR)
- 배경 사진 업로드 후 배경 컨트롤의 **🔎 수치 OCR** 을 누릅니다. (로컬 서버 실행 필요)
- 사진 속 숫자를 자동 인식하여, **"가로×세로" 쌍은 한 번에 방 생성**, 개별 숫자는 두 개를 골라 방을 만들 수 있습니다.
- 인식값 단위(mm/cm/m)를 선택할 수 있고, 생성 후 속성창에서 치수를 보정합니다.
- ※ 사진 화질·글꼴에 따라 정확도가 달라지는 보조 기능입니다. 정밀 작도는 **축척보정 + 따라 그리기** 를 권장합니다.

### 6. 3D 공간 보기 (1인칭 FPS 시점)
- 각 도면 툴바의 **🧊 3D** 버튼을 누르면 해당 집이 3D로 렌더링됩니다.
- 방은 **바닥 + 반투명 벽**으로, 가구는 **실제 가로×깊이×높이 크기의 박스**로 표시되어 공간을 얼마나 차지하는지 한눈에 확인할 수 있습니다.
- 가구 위에는 이름과 치수(W×D×H) 라벨이 떠 있으며, **충돌 중인 가구는 붉게**, 방 밖 가구는 주황빛으로 표시됩니다.
- **조작 (FPS 게임 방식)**
  - 화면 클릭 → 마우스로 시점을 자유롭게 둘러봅니다 (1인칭, 눈높이 1.6m)
  - **W / A / S / D** (또는 방향키) 이동 · **Shift** 달리기
  - **Space** 위로 · **C** 아래로 (공중에서 조감 가능)
  - **Esc** 마우스 해제, 한 번 더 **Esc** 로 닫기
- 상단 바에서 **벽 높이(mm)** 와 **벽 투명도**를 조절할 수 있고, "↺ 시작 위치로"로 처음 시점으로 돌아갑니다.
- 가구 높이는 등록 폼의 "높이" 칸 또는 선택 후 속성창의 "높이" 필드로 수정합니다.

### 7. 배치도 내보내기 (PNG / PDF)
- 상단 **🖼 PNG** / **📄 PDF** 로 두 집의 배치도를 한 장에 저장합니다.
- 방·가구·치수·면적 합계·충돌 경고가 포함되며, 화면 확대/이동과 무관하게 전체 내용이 항상 담깁니다.

### 8. 기타
- 상단 **단위** 로 mm / cm / m 전환.
- 도면별 **격자**, **+ / −** 확대·축소. 빈 영역 드래그로 화면 이동(패닝).
- **⇄ 축척맞춤**: 두 도면의 배율(1m당 화면 픽셀)을 일치시켜 같은 기준으로 비교합니다. 왼쪽 툴바의 "⇄ 오른쪽에 축척맞춤"은 왼쪽을 오른쪽 배율에, 오른쪽 툴바의 "⇄ 왼쪽에 축척맞춤"은 오른쪽을 왼쪽 배율에 맞춥니다.
- **🔄 A↔B 맞바꾸기**: 상단 버튼으로 두 도면의 내용(배경·방·가구·배율)을 통째로 교환합니다. 패널 이름은 그대로 유지되므로, 현재 집/이사갈 집 슬롯을 잘못 채웠을 때 한 번에 바로잡을 수 있습니다(다시 누르면 원복).
- 작업은 자동 저장(브라우저 localStorage)되며, **저장 / JSON 내보내기 / 불러오기** 로 백업·공유할 수 있습니다.
- 선택 후 **Delete** 키로 삭제, **Esc** 로 선택/팝업 해제.

## 단축키
| 키 | 동작 |
|---|---|
| Delete / Backspace | 선택 도형 삭제 |
| Esc | 비교 팝업 닫기 / 비교 모드 해제 / 선택 해제 |

## 파일 구성
- `index.html` — 화면 구조
- `styles.css` — 스타일
- `app.js` — 2D 도면·가구 로직
- `viewer3d.js` — 3D 공간 뷰어(1인칭 FPS)
- `server.js` — 의존성 없는 로컬 정적 서버(Node)
- `start.ps1` — 서버 실행 + 브라우저 열기
- `vendor/konva.min.js` — 2D 캔버스 라이브러리(로컬 포함)
- `vendor/three.min.js` — 3D 렌더링 라이브러리(로컬 포함)
- `vendor/jspdf.umd.min.js` — PDF 생성 라이브러리(로컬 포함)
- OCR 엔진(Tesseract.js)은 최초 사용 시 인터넷에서 로드됩니다.
