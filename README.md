# 🏠 아파트 평면도 비교 · 가구 배치 시뮬레이터

이사를 앞두고 **현재 집(A)** 과 **이사갈 집(B)** 의 평면도를 나란히 놓고 비교하면서, 가지고 있는 가구가 새 집에 들어가는지 미리 배치해 볼 수 있는 로컬 웹 애플리케이션입니다.

- 설치가 필요 없습니다. 크롬 브라우저만 있으면 실행됩니다.
- 모든 데이터는 내 PC(브라우저)에만 저장됩니다. 외부 서버로 전송되지 않습니다.
- 2D 도면 편집과 **3D 1인칭(FPS) 공간 둘러보기**를 모두 지원합니다.

## 1. 실행 방법

### 방법 A — 로컬 서버로 실행 (권장, 모든 기능 사용 가능)

1. 프로젝트 폴더의 `start.ps1` 을 우클릭 → **"PowerShell에서 실행"** 을 선택합니다. (Node.js 필요)
2. 자동으로 로컬 서버가 시작되고 크롬에서 `http://localhost:8765/` 가 열립니다.
3. **OCR(사진 치수 자동인식)** 은 브라우저 보안 정책상 이 방식에서만 동작합니다.

서버를 종료해도 작업 내용은 브라우저에 저장되어 있으므로 사라지지 않습니다. (자세한 내용은 [13. 저장과 백업](#13-저장과-백업) 참고)

### 방법 B — 파일 직접 열기 (간편 실행)

- `index.html` 을 더블클릭하거나 크롬으로 엽니다.
- OCR을 제외한 모든 기능(2D 편집, 3D 보기, 충돌 경고, 내보내기 등)이 동작합니다.

### 방법 C — 안드로이드 앱 (APK)

- `build-apk.ps1` 로 빌드한 `app-debug.apk` 를 휴대폰에 설치하면 동일한 앱이 모바일에서 동작합니다.
- **터치 조작 지원**: 3D 보기에서 왼쪽 화면의 가상 조이스틱으로 이동하고, 오른쪽 화면을 드래그해 시점을 돌립니다. 우측 하단 ▲/▼ 버튼으로 위/아래 이동이 가능합니다.
- 좁은 화면에서는 상단의 **☰ 가구** / **⚙ 속성** 버튼으로 사이드 패널을 열고 닫습니다. (가로 모드 고정)
- PNG/PDF/JSON 내보내기는 기기의 **다운로드 폴더**에 저장되고, "📷 도면사진"은 갤러리에서 선택합니다.
- 데이터는 앱 내부 저장소(localStorage)에 자동 저장됩니다.
- 설치 시 "출처를 알 수 없는 앱" 허용이 필요합니다(직접 서명한 개발용 APK이기 때문입니다).

빌드 방법(최초 1회 도구 준비 후):

```powershell
# 필요 도구: %LOCALAPPDATA%\AndroidBuild\ 아래 JDK 17, Gradle 8.7, Android SDK 34
.\build-apk.ps1
# → android\app\build\outputs\apk\debug\app-debug.apk
```

> ⚠️ 두 방식은 브라우저 저장소(origin)가 분리됩니다. 방법 A로 작업한 내용은 방법 B에서 보이지 않습니다. 작업을 옮길 때는 **⬇ JSON 저장 → ⬆ JSON 열기** 를 이용해 주세요. 한 가지 방식으로 계속 사용하시길 권장합니다.

### 예시 데이터로 시작하기

처음 사용법을 익힐 때는 주소 끝에 `?demo=1` 을 붙여 주세요. 예시 방·가구가 채워진 상태로 시작됩니다.

```
http://localhost:8765/?demo=1
```

## 2. 화면 구성

```
┌────────────────────────── 상단 바 ──────────────────────────┐
│ 단위 · 방 비교 · A↔B 맞바꾸기 · PNG · PDF · 저장 · JSON 저장/열기 · 초기화 │
├──────────┬──────────────────────────────┬──────────────────┤
│  가구     │   현재 집 (A)   │  이사갈 집 (B)   │   속성 패널        │
│  라이브러리│   (도면 캔버스)  │  (도면 캔버스)    │   (선택 도형 편집)  │
│  도움말   │                 │                 │   면적 요약        │
└──────────┴──────────────────────────────┴──────────────────┘
```

| 영역 | 설명 |
|---|---|
| **상단 바** | 단위 전환(mm/cm/m), 방 비교, A↔B 맞바꾸기, 내보내기, 저장/불러오기, 초기화 |
| **좌측 사이드바** | 가구 라이브러리(등록·수정·삭제·배치), 도움말 |
| **중앙 도면 2개** | 왼쪽 = 현재 집(A), 오른쪽 = 이사갈 집(B). 각 도면 위에 전용 툴바가 있습니다 |
| **우측 속성 패널** | 선택한 방/가구의 이름·치수·회전·색상 편집, 두 집의 방 면적 합계(㎡/평) |

각 도면 툴바 버튼: **+ 방** · **+ 가구** · **📷 도면사진** · **축척보정** · **− / +** (확대·축소) · **⇄ 축척맞춤** · **격자** · **🧊 3D**

도면 이름("현재 집 (A)" 등)은 클릭해서 자유롭게 바꿀 수 있습니다.

## 3. 평면도(방) 만들기

방법은 세 가지이며, 병행해서 사용할 수 있습니다.

### ① 치수를 직접 입력

1. 도면 툴바의 **+ 방** 을 누릅니다.
2. 방 이름 → 가로 → 세로 순으로 입력하면 사각형 방이 생성됩니다. (현재 선택된 단위 기준)
3. 생성 후 드래그로 위치를 잡고, 우측 속성 패널에서 치수를 정밀하게 수정합니다.

방에는 이름·치수·면적(㎡)이 항상 표시되며, 우측 "면적 요약"에 집 전체 합계가 ㎡와 평으로 집계됩니다.

### ② 평면도 사진 위에 따라 그리기 (정밀 작도에 권장)

1. **📷 도면사진** 으로 평면도 이미지를 업로드합니다.
2. **축척보정** 으로 사진의 실제 비율을 맞춥니다. ([9번 항목](#9-사진-도면-활용--배경--축척보정--ocr) 참고)
3. 사진 위에 **+ 방** 으로 방을 만들고 모서리 핸들로 사진의 방 윤곽에 맞춥니다.

### ③ 사진 치수 OCR 자동인식

사진 속 치수 숫자를 자동으로 읽어 방을 생성합니다. ([9번 항목](#9-사진-도면-활용--배경--축척보정--ocr) 참고)

## 4. 가구 등록과 배치

### 가구 등록 (좌측 가구 라이브러리)

1. 이름(예: 퀸 침대), **가로 × 세로 × 높이**, 색상을 입력하고 **+ 가구 등록** 을 누릅니다.
2. **높이**는 3D 보기에서 사용됩니다. 비워 두면 이름으로 추정한 기본값이 자동 적용됩니다. (침대 500mm, 소파 850mm, 냉장고 1,800mm, 옷장 2,100mm, 책상·식탁 740mm 등)

> 💾 **한 번 등록한 가구는 영구 저장됩니다.** 브라우저를 닫아도, "초기화"를 눌러도, 다른 JSON 파일을 불러와도 카탈로그는 그대로 유지되어 다음 이사 때도 재사용할 수 있습니다.

목록의 각 가구 항목 버튼:

| 버튼 | 동작 |
|---|---|
| **→A / →B** | 해당 도면 중앙에 가구 추가 |
| **✎** | 등록 정보(이름·치수·색상) 수정 |
| **✕** | 카탈로그에서 삭제 (이미 배치된 가구는 유지됨) |

### 가구 배치 (3가지 방법)

1. **드래그 앤 드롭** — 목록의 가구를 도면 위 원하는 자리로 끌어다 놓습니다. 방 비교 팝업의 방에도 동일하게 드롭할 수 있습니다.
2. **→A / →B 버튼** — 해당 도면 중앙에 추가됩니다.
3. **도면 툴바의 + 가구** — 등록된 가구 목록에서 골라 추가합니다.

배치된 가구는 등록된 실제 치수로 그려지며, 방과 달리 **크기가 고정**됩니다(실물 크기이므로). 치수를 바꾸려면 선택 후 속성 패널에서 수정합니다.

## 5. 도형 편집 — 이동 · 회전 · 다중 선택

| 동작 | 방법 |
|---|---|
| **선택** | 도형 클릭 |
| **이동** | 도형 드래그 |
| **360° 자유 회전** | 선택 후 위쪽 회전 핸들을 돌립니다. 0°/90°/180°/270° 근처에서 자동 스냅됩니다 |
| **정밀 회전** | 속성 패널의 회전 슬라이더, **+90° / +45°** 버튼 |
| **방 크기 조절** | 방 선택 후 모서리·변 핸들 드래그 (가구는 실측 고정) |
| **다중 선택** | **Shift+클릭** (또는 Ctrl/⌘+클릭)으로 여러 도형을 추가 선택 |
| **함께 이동** | 다중 선택 상태에서 아무 도형이나 드래그하면 전체가 함께 이동 |
| **삭제** | 선택 후 **Delete** 키, 또는 속성 패널의 삭제 버튼 (다중 선택 시 일괄 삭제) |
| **선택 해제** | 빈 곳 클릭 또는 **Esc** |
| **화면 이동(패닝)** | 도면의 빈 영역을 드래그 |

속성 패널에서는 이름, 가로/세로(가구는 높이 포함), 회전 각도, 색상을 수치로 편집할 수 있습니다.

## 6. 가구 겹침 · 방 이탈 경고

가구를 드래그하거나 회전하는 **즉시** 실시간으로 검사합니다. 회전된 사각형끼리도 정확하게 판정합니다(SAT 알고리즘).

| 표시 | 의미 |
|---|---|
| 🔴 **빨강 점선 + 붉은 채움** | 다른 가구와 겹침(충돌) |
| 🟠 **주황 점선** | 가구가 방 밖으로 벗어남(전부 또는 일부) |
| 도면 우측 상단 배지 | `⚠ 충돌 N개 · 방 이탈 M개` 실시간 집계, 문제가 없으면 `✓ 충돌 없음` |

이 경고는 3D 보기와 PNG/PDF 내보내기에도 동일하게 반영됩니다.

## 7. 두 집 방 비교 시뮬레이션

"지금 안방에 있는 가구가 새 집 안방에 들어갈까?"를 확인하는 핵심 기능입니다.

1. 상단 **📐 방 비교 (방 2개 선택)** 를 누릅니다.
2. 안내 배너가 뜨면 도면에서 비교할 방 2개를 차례로 클릭합니다. (예: A의 안방 → B의 안방)
3. 비교 팝업이 열립니다.

### 비교 팝업에서 할 수 있는 것

- **같은 축척으로 나란히 보기** — 두 방이 동일한 배율로 표시되어 크기 차이가 직관적으로 보입니다.
- **수치 비교 카드** — 면적(㎡/평), 가로/세로 차이가 표시됩니다. 증가는 초록, 감소는 빨강입니다.
- **겹쳐보기** — 상대 방의 외곽선을 점선으로 겹쳐서 어느 방향으로 얼마나 큰지/작은지 확인합니다.
- **가구 트레이** — 등록된 가구를 좌우 방에 넣고(**◀넣기 / 넣기▶** 버튼 또는 드래그 앤 드롭) 이동·회전하며 실제로 들어가는지 시뮬레이션합니다. 충돌 경고도 동일하게 동작합니다.

팝업은 **닫기** 버튼 또는 **Esc** 로 닫습니다. 팝업 안에서 배치한 가구는 시뮬레이션용이며 원본 도면에는 영향을 주지 않습니다.

## 8. 3D 공간 보기 (1인칭 FPS)

도면 툴바의 **🧊 3D** 버튼을 누르면 해당 집이 3차원 공간으로 렌더링됩니다.

### 표시 내용

- **방** — 바닥 슬래브 + 반투명 벽. 벽 높이는 기본 2,400mm이며 조절 가능합니다.
- **가구** — 등록한 **가로 × 깊이 × 높이** 실측 크기의 박스로 표시되어, 각 가구가 공간을 얼마나 차지하는지 눈높이에서 체감할 수 있습니다.
- **라벨** — 방과 가구 위에 이름·치수(W×D×H)가 떠 있습니다.
- **경고 연동** — 겹친 가구는 붉게, 방을 벗어난 가구는 주황빛으로 표시됩니다.

### 조작 방법 (FPS 게임 방식)

| 조작 | 동작 |
|---|---|
| **화면 클릭** | 1인칭 시점 시작 (마우스 잠금) |
| **마우스 이동** | 시점 회전 (둘러보기, 눈높이 1.6m) |
| **W / A / S / D** (또는 방향키) | 앞 / 왼쪽 / 뒤 / 오른쪽 이동 |
| **Shift** | 달리기 (이동 속도 2배 이상) |
| **Space** | 위로 상승 (공중에서 조감 가능) |
| **C** | 아래로 하강 |
| **Esc** | 마우스 잠금 해제 → 한 번 더 누르면 3D 닫기 |

### 상단 조절 바

- **벽 높이(mm)** — 실제 천장고에 맞게 조절합니다. 도면별로 저장됩니다.
- **벽 투명도** — 벽 너머가 보이는 정도를 조절합니다.
- **↺ 시작 위치로** — 처음 시점으로 돌아갑니다.
- **닫기 (Esc)** — 2D 화면으로 복귀합니다.

가구 높이는 등록 폼의 "높이" 칸에서, 이미 배치된 가구는 선택 후 속성 패널의 "높이" 필드에서 수정합니다.

## 9. 사진 도면 활용 — 배경 · 축척보정 · OCR

### 배경 사진 올리기

1. 도면 툴바의 **📷 도면사진** 을 눌러 평면도 이미지를 업로드합니다. (JPG/PNG 등)
2. 사진이 도면 배경에 깔리고, 좌측 하단에 배경 컨트롤(투명도 슬라이더, 수치 OCR, 배경 제거)이 나타납니다.
3. 배경 사진은 드래그로 위치를 옮길 수 있습니다.

> ⚠️ 사진 업로드는 반드시 **📷 도면사진** 버튼을 사용해 주세요. 상단의 "⬆ JSON 열기"는 저장해 둔 프로젝트 파일(.json) 복원 전용입니다.

### 축척보정 (사진을 실제 비율로)

1. **축척보정** 버튼을 누릅니다.
2. 사진에서 **실제 길이를 알고 있는 두 점**을 차례로 클릭합니다. (예: "4500"이라고 적힌 벽의 양 끝)
3. 그 구간의 실제 길이를 입력하면 사진이 실제 축척으로 맞춰집니다.
4. 이제 사진 위에 방을 만들면 실측 그대로 따라 그릴 수 있습니다.

### 수치 OCR 자동인식 (로컬 서버 실행 시에만)

1. 배경 컨트롤의 **🔎 수치 OCR** 을 누릅니다. 최초 1회는 OCR 엔진을 인터넷에서 내려받습니다.
2. 사진 속 숫자를 인식하여 결과 창에 표시합니다.
   - **"가로×세로" 쌍** (예: `4500×3200`)으로 인식된 항목은 **방 생성** 버튼 한 번으로 방이 만들어집니다.
   - 개별 숫자는 칩 형태로 나열되며, **두 개를 클릭(가로 → 세로)** 해서 방을 만들 수 있습니다.
3. 인식값의 단위(mm/cm/m)는 창 상단에서 선택합니다.

> ※ 사진 화질·글꼴에 따라 정확도가 달라지는 보조 기능입니다. 잘못 읽힌 값은 무시하고, 생성 후 속성 패널에서 보정해 주세요. 정밀 작도에는 **축척보정 + 따라 그리기**를 권장합니다.

## 10. 화면 배율 · 격자 · 축척맞춤

- **− / +** — 도면을 축소/확대합니다. (도형의 실제 치수는 변하지 않습니다)
- **격자** — 1m 간격 격자를 켜고 끕니다. 5m마다 굵은 선이 표시됩니다.
- **⇄ 축척맞춤** — 두 도면의 화면 배율(1m당 픽셀)을 일치시켜 **같은 기준으로 크기를 비교**할 수 있게 합니다.
  - 왼쪽 도면의 "⇄ 오른쪽에 축척맞춤" → 왼쪽 배율을 오른쪽에 맞춤
  - 오른쪽 도면의 "⇄ 왼쪽에 축척맞춤" → 오른쪽 배율을 왼쪽에 맞춤
- **단위 전환** — 상단 바에서 mm/cm/m 을 바꾸면 모든 치수 표기와 입력 기준이 함께 바뀝니다.

## 11. A↔B 맞바꾸기

상단 **🔄 A↔B 맞바꾸기** 버튼은 두 도면의 **내용 전체(배경 사진·방·가구·배율·격자 설정)** 를 통째로 교환합니다.

- 패널 이름("현재 집 (A)" 등)과 위치는 그대로 유지됩니다.
- 실수로 이사갈 집 도면을 현재 집(A) 쪽에 만들어 버렸을 때 한 번에 바로잡을 수 있습니다.
- 다시 누르면 원래대로 돌아옵니다.

## 12. 내보내기 (PNG / PDF)

상단 **🖼 PNG** / **📄 PDF** 버튼으로 두 집의 배치도를 한 장의 문서로 저장합니다.

- 두 도면이 나란히 배치되고 방·가구·치수·면적 합계·충돌 경고 요약이 포함됩니다.
- 화면을 얼마나 확대/이동해 두었는지와 관계없이 **항상 전체 내용이 담기도록** 자동 맞춤됩니다.
- PDF는 A4 가로 방향으로 생성됩니다. 이사 업체 공유나 인쇄에 활용해 주세요.

## 13. 저장과 백업

| 기능 | 설명 |
|---|---|
| **자동 저장** | 모든 편집은 브라우저 저장소(localStorage)에 자동 저장됩니다. 서버를 꺼도, 브라우저를 닫아도, PC를 재시작해도 유지됩니다 |
| **💾 저장** | 자동 저장을 기다리지 않고 즉시 저장합니다 |
| **⬇ JSON 저장** | 현재 작업 전체를 `.json` 파일로 내려받습니다. **백업·PC 간 이동용으로 권장**합니다 |
| **⬆ JSON 열기** | 저장해 둔 `.json` 프로젝트를 복원합니다. 파일 속 가구도 카탈로그에 합쳐집니다 |
| **초기화** | 도면·가구 배치를 모두 지웁니다. 단, **가구 카탈로그는 유지**됩니다 |

주의사항:

- 저장은 **주소(origin) 단위**로 분리됩니다. `http://localhost:8765` 와 `file://` 로 연 경우 서로 다른 저장소를 사용합니다.
- 브라우저의 "인터넷 사용 기록 삭제"에서 사이트 데이터를 지우면 저장 내용도 삭제됩니다. 중요한 작업은 **⬇ JSON 저장**으로 파일 백업을 해 두시기 바랍니다.

## 15. 자주 묻는 질문 (FAQ)

**Q. 서버(start.ps1)를 꺼도 작업 내용이 남아 있나요?**
네. 데이터는 서버가 아니라 브라우저(localStorage)에 저장되므로 서버 종료·PC 재시작과 무관하게 유지됩니다. 서버는 파일을 제공하는 역할만 합니다.

**Q. 평면도 사진을 올리려는데 "JSON 파일이 아닙니다" 오류가 납니다.**
상단 "⬆ JSON 열기"가 아니라 각 도면 툴바의 **📷 도면사진** 버튼으로 올려 주세요.

**Q. OCR 버튼이 동작하지 않습니다.**
OCR은 `index.html` 을 직접(더블클릭) 연 경우 브라우저 보안 정책으로 차단됩니다. `start.ps1` 로 서버를 띄워 `http://localhost:8765/` 로 접속해 주세요. 최초 1회는 인터넷 연결도 필요합니다.

**Q. 3D에서 가구 높이가 이상합니다.**
높이를 입력하지 않은 가구는 이름으로 추정한 기본값이 적용됩니다. 가구를 선택한 뒤 우측 속성 패널의 "높이" 필드에서 실제 값으로 수정하거나, 라이브러리의 ✎ 버튼으로 등록 정보를 고쳐 주세요.

**Q. 가구는 왜 크기 조절 핸들이 없나요?**
가구는 실물 치수를 그대로 반영하는 것이 목적이므로 캔버스에서의 자유 변형을 막아 두었습니다. 치수 변경은 속성 패널에서 수치로 해 주세요.

**Q. 인터넷 없이 사용할 수 있나요?**
네. 렌더링 라이브러리(Konva.js, Three.js, jsPDF)가 모두 로컬에 포함되어 있어 오프라인에서 동작합니다. 예외적으로 OCR 엔진(Tesseract.js)만 최초 사용 시 인터넷에서 로드됩니다.

## 16. 파일 구성

| 파일 | 역할 |
|---|---|
| `index.html` | 화면 구조 |
| `styles.css` | 스타일 (다크 테마) |
| `app.js` | 2D 도면·가구·비교·내보내기·저장 로직 |
| `viewer3d.js` | 3D 공간 뷰어 (1인칭 FPS) |
| `server.js` | 의존성 없는 로컬 정적 서버 (Node.js) |
| `start.ps1` | 서버 실행 + 브라우저 열기 스크립트 |
| `vendor/konva.min.js` | 2D 캔버스 라이브러리 (로컬 포함) |
| `vendor/three.min.js` | 3D 렌더링 라이브러리 (로컬 포함) |
| `vendor/jspdf.umd.min.js` | PDF 생성 라이브러리 (로컬 포함) |
| `android/` | 안드로이드 WebView 앱 프로젝트 |
| `build-apk.ps1` | APK 빌드 스크립트 (웹 자산 복사 + Gradle 빌드) |

### 기술 메모

- 내부 기준 단위는 **mm**, 도형 중심 좌표는 월드 미터(m)로 저장하며, 화면은 도면별 배율(1m당 px)로 렌더링합니다.
- 충돌 판정은 회전 사각형(OBB) 간 **분리축 정리(SAT)**, 방 이탈은 볼록다각형 내부 판정을 사용합니다.
- 3D는 Three.js 기반이며 2D의 좌표·회전을 그대로 사용합니다 (x→X, y→Z, 회전은 Y축).
